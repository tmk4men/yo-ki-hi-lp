# Yo-Ki-Hi LP 要件定義書

> 作成日: 2026-05-04
> 対象: 柔道整復師（整骨院・整体院・治療院・サロン関係者）向け 1ページ LP + パスワード分岐認証
> 出典: クライアント送付モック (`https://yo-ki-hi-mock.netlify.app/`) の要件定義およびユーザー指示

---

## 1. 目的

整骨院・整体院・治療院・サロンの **導入検討者および現場スタッフ** に対し、Yo-Ki-Hi（骨盤ベルト／インナーパンツ）の **理論・検査方法・装着方法・使い分け・EMS 活用・購入導線** を 1ページで完結させる業務用導入支援サイトを提供する。

現状は動画・LINE・購入ページ・パスワード案内が分散しており、導入検討者ほど理解と購入が難しい。本サイトはこれを集約し、セミナー後に共有できる単一 URL として機能させる。

## 2. ターゲット

| 区分 | 想定 |
| --- | --- |
| 主 | 整骨院、整体院、治療院、サロン経営者・スタッフ |
| 主 | 船井総研などのセミナー経由で関心を持った事業者 |
| 副 | 導入後に患者へ説明する現場スタッフ |
| 副 | 平森先生、太田先生のコミュニティ参加者（塾生） |

一般消費者向けの美容LPではない。あくまで **業務用マニュアル兼販売支援ページ** として設計する。

## 3. スコープ（MVP / 5/24 セミナー版）

### 含めるもの（MVP）

- 1ページ構成のトップページ
- ファーストビューの 6 入口カード
- 理論セクション（骨盤を立てる考え方、Before/After、4 軸の説明）
- 検査方法セクション（5 ステップ、補助動画、Before/After 比較）
- 正しい装着方法セクション（位置・ベクトル、図解 2 枚）
- 使い分けセクション（ベルト／インナー比較表、シーン別チップ）
- EMS 活用セクション（Before / Process / After フロー、図解、補助動画）
- 商品情報セクション（ベルト／インナー 2 商品の仕様カード、CTA バンド）
- パスワード認証ページ（共通入力欄 + 利用案内）
- パスワード判定により遷移する **3 種類の購入ページ**（導入店／平森塾生／太田塾生）

### Phase 2 に回すもの

```

  #wearing                正しい装着方法
  #selector               使い分け
  #ems                    ベルト×EMS
  #products               商品情報
/purchase-login           パスワード認証
/purchase-shop            購入: 導入店/一般導入者
/purchase-hiramori        購入: 平森先生の塾生
/purchase-ota             購入: 太田先生の塾生
```

### 4.1 トップページ セクション要件

| # | セクション | 主な要件 |
| --- | --- | --- |
| 01 | First View | "知りたい内容へ、すぐ移動できます" を主見出しに、6 入口カード（理論・検査・装着・使い分け・EMS・商品情報）を配置。スマホでも押しやすい大きさ |
| 02 | 理論 | "骨盤を立てることが姿勢づくりの起点" を中心軸に、Before/After カードと 4 軸（可・体・呼・美）を漢字 1 字記号で表現 |
| 03 | 検査方法 | 5 ステップ（後屈／肩／体幹／ガッツポーズ／装着後反復）。動画は補助、手順とチェックポイントを主役に。Before / After 体感の3列比較を card に |
| 04 | 正しい装着方法 | 位置とベクトルが効果を決めることを伝える。図解2枚（装着・確認方法／正しい装着位置）。最終版では実物・モデル写真への差し替えを想定 |
| 05 | 使い分け | "施術後・日常はベルト／運動・長時間はインナー" を比較表で。シーン別チップ（就寝→ベルト、スポーツ→インナー など）を5枚 |
| 06 | EMS | Before → EMS+ベルト → After のフロー。10分活用の流れを補助動画と全身変化図解で |
| 07 | 商品情報 | ベルト・インナーの2カードを並列。価格・サイズ・素材・推奨シーン。Choose Your Style CTAバンドで購入認証へ |

### 4.2 パスワード認証ページ要件

- ヘッダー / フッターはトップと統一
- 主見出し: 「購入はこちら」
- 説明: 「案内されたパスワードを入力してください。パスワードに応じた購入ページへお進みいただけます。」
- フォーム: パスワード単一入力欄、送信ボタン「購入ページへ進む」
- 案内パネル: 対象 3 区分（導入店/一般導入者・平森先生の塾生・太田先生の塾生）を箇条書き
- 結果領域 (`aria-live="polite"`): 入力前は案内テキスト、誤りは赤系フィードバック、正しい場合はそのまま該当ページに遷移
- 補足文: 「パスワードをお持ちでない場合は、案内元へお問い合わせください。」

### 4.3 3 つの購入ページ要件（共通レイアウト・差分は内容のみ）

| ページ | 対象 | 表示する主な内容 |
| --- | --- | --- |
| `/purchase-shop` | 導入店 / 一般導入者 | 導入店向け価格区分の購入リンク、納品手続き・問い合わせ窓口 |
| `/purchase-hiramori` | 平森先生の塾生 | 塾生限定価格、平森先生コミュニティ向け案内、サポートチャネル |
| `/purchase-ota` | 太田先生の塾生 | 塾生限定価格、太田先生コミュニティ向け案内、サポートチャネル |

各ページは MVP では **プレースホルダ（実 URL・実価格は後日差し替え）** とし、`Shopify URL 等を埋め込むまでの仮置き領域` を内部コメントで明示する。

## 5. 機能要件

### 5.1 パスワード分岐認証

- フォーム送信時に入力値を `passwordRoutes` マップで照合し、対応する購入ページへ `location.assign()` で遷移
- パスワード未入力 → ガイダンス表示（青系トーン）
- パスワード不一致 → エラー表示（赤系トーン）+ 入力欄シェイクアニメ
- パスワード一致 → 「○○ の購入ページへ進みます」を success トーンで表示し 600ms 後に遷移

### 5.2 セキュリティに関する明記

- **MVP では JS 内に共通プレースホルダパスワードを保持する**（`shop-demo` / `hiramori-demo` / `ota-demo`）。
- 本番運用では **必ずサーバー側でパスワード判定** を行うこと。フロントエンドにパスワード・購入URLを直接書くと、ソース閲覧で漏洩する。
- 本番移行時の推奨: Cloudflare Worker / Netlify Function / Vercel Edge Function などで判定 + 署名付き URL リダイレクト。

### 5.3 アクセシビリティ

- すべての画像に `alt`
- ナビゲーションに `aria-label`
- 結果通知領域に `aria-live="polite"`
- フォーカスリング保持
- キーボードのみで全操作可能

### 5.4 アニメーション

- スクロールリビール: IntersectionObserver で `.reveal` を `is-visible` 化、各要素に少しずつ遅延（最大 260ms）
- ホバー: ボタン / カードの translateY -1〜-4px と shadow 強化
- ナビ下線: scaleX(0) → scaleX(1) のホバートランジション
- `prefers-reduced-motion: reduce` を尊重し、動きを削減

### 5.5 OGP / SEO

- `og:title`, `og:description`, `og:type=website`, `og:url`, `og:image`
- Twitter Card: `summary_large_image`
- 1200×630 OG 画像を用意（SVG 由来で軽量化）
- description は 120 文字程度
- favicon: SVG（フォールバック ICO）

## 6. デザイン要件

### 6.1 トーン & マナー

モック踏襲。和洋折衷の **ナチュラル × 上品** トーン。

| 役割 | 値 |
| --- | --- |
| 背景 | `#f5efe7`（ベージュ）+ 上下グラデ + 左右の柔らかな blob |
| サーフェス | `rgba(255, 252, 247, 0.86)` + `backdrop-filter: blur(16px)` |
| 主アクセント | `#8d4f3f`（赤茶）／深 `#6f3d2f` |
| 補助 | `#6f7f71`（セージ）／`#b68d49`（ゴールド） |
| 墨 | `#30251f` |
| 影 | `0 28px 80px rgba(88, 63, 44, 0.12)` |

### 6.2 タイポグラフィ

- 見出し: `"Iowan Old Style", "Palatino Linotype", "Yu Mincho", "Hiragino Mincho ProN", serif`（明朝系）
- 本文: `"Avenir Next", "Hiragino Sans", "Yu Gothic", sans-serif`
- 見出し letter-spacing 0.01〜0.08em。小見出し eyebrow は `text-transform: uppercase` + 0.16em 字間
- Inter / Roboto / Arial など generic AI font は **使わない**

### 6.3 装飾の方針

- 漢字一字（可・体・呼・美）の小型バッジでアイコンの代用。SVG アイコンセット濫用を回避
- カード角丸 `1.4〜1.8rem`、ふんわりした blob を背景に 1〜2 個のみ
- 紫グラデ on 白背景は **禁止**

### 6.4 レスポンシブ

| ブレークポイント | 主な変化 |
| --- | --- |
| `≥ 1080px` | 2〜4 カラムグリッド、ヘッダー横並び |
| `≤ 1080px` | grid を順次 1 カラム化、hero-points は 3 列維持 |
| `≤ 820px` | ヘッダー縦並び、product-grid 等を 2 列化、比較表を縦化 |
| `≤ 640px` | 1 カラム / フル幅ボタン、見出しを `clamp` で縮小、CTA 縦積み |

## 10. 技術構成

- 静的 HTML / CSS / Vanilla JS
- ビルドツールなし（プレーンに保ちネット環境の弱い現場でも軽快に動作）
- ホスティングは Netlify / Cloudflare Pages を想定（`_redirects` 設定で `/purchase-login` 等のクリーン URL を確保）
