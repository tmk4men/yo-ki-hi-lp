Requirements with Screenshots

骨盤ベルト/インナー関連サイト 要件定義

この資料は、制作中のモック画面と要件を対応させた確認用ページです。 セミナー後に先生方へ共有し、理論・検査・装着・使い分け・EMS活用・購入導線を短時間で理解してもらうことを目的にしています。

Target

治療院関係者向け

整骨院、整体院、治療院、サロン、セミナー参加者、現場スタッフが見る業務用の導入支援ページ。

MVP

5/24セミナー用

1ページサイト、図解中心、スマホ対応、購入認証ページまでを初期公開範囲にする。

Purchase

パスワード分岐

購入ボタンから認証ページへ進み、パスワードに応じて3つの購入ページへ自動分岐する。

画面別要件

モック画面と要件の対応

01 / First View

最初に6つの入口を大きく見せる

サイトに入った瞬間に、どの情報が見られるか分かる構成。 一般向けLPではなく、導入検討者が必要な情報へすぐ移動できる業務用導線にする。

  • 理論、検査方法、正しい装着方法、使い分け、EMS、商品情報へ直接移動
  • ボタンはスマホでも押しやすい大きさ
  • セミナー後にURLを共有しても迷わない構成

02 / Check

効果実感のための検査を主役にする

導入・販売の肝は、商品説明よりも変化を見せる検査。 動画は補助として置き、手順とチェックポイントがすぐ分かる形を優先する。

  • 腰の後屈、肩・腕、体幹、ガッツポーズでの安定性を初期版の主軸にする
  • Before/Afterで同じ検査を行う流れを明確にする
  • 動画を見る負担を減らし、図解中心で理解できるようにする

03 / Wearing

正しい装着位置を独立して説明する

ベルトは位置とベクトルが重要。 検査方法とは別に、装着位置、締める強さ、NG例、確認ポイントを見せる。

  • AIイラストだけでなく、最終版では実物写真・モデル着用写真を入れる
  • 高すぎる位置、締めすぎ、ズレやすいケースを補足する
  • 販売時や現場説明時にそのまま見せられる内容にする

04 / Select

ベルトとインナーの提案基準を比較表にする

どちらが良いかではなく、生活スタイルに応じて提案できるようにする。 施術後・日常はベルト、運動・長時間はインナーという整理を見せる。

  • ベルトは初回体感、短時間、施術後の日常サポート向き
  • インナーは運動、長時間、ズレにくさ、継続使用向き
  • 仕事中はインナー、帰宅後や就寝時はベルトなど併用提案も可能

05 / EMS

EMSとの併用をメニュー提案につなげる

ベルト単体販売ではなく、治療院側の施術メニュー化・単価アップ提案につなげるセクション。 10分活用モデルとBefore/After検査を見せる。

  • Before → EMS+ベルト → After の流れを示す
  • 施術現場での使い方、患者さんへの説明例を追加できる構成
  • 動画は補助、図解とフローを主役にする

06 / Purchase

購入ボタンからパスワード認証へ進む

初期版では会員登録やメールログインは不要。 購入意欲が高い人が迷わないよう、パスワードだけで購入ページへ分岐する。

  • 導入店/一般導入者、平森先生の塾生、太田先生の塾生の3区分
  • 実パスワードと購入URLは本番ではサーバー側で判定する
  • 代理店・大口販売用は初期版では外し、必要なら後から追加

MVP / Phase 2

5/24に間に合わせるための切り分け

MVPで入れる

  • 1ページサイト
  • ファーストビューの6入口
  • 理論、検査方法、正しい装着方法
  • ベルト/インナー使い分け
  • EMS活用
  • 価格・サイズ・購入
  • 購入ボタンから認証ページへ遷移
  • パスワードごとの購入ページ分岐

Phase 2に回す

  • メールログイン、会員登録
  • 事業者申請、治療院名チェック
  • Googleマップ照合
  • 購入者ログ管理
  • LINE連携、自動返信
  • 姿勢解析、MediaPipe連携
  • 導入院別ページ、FAQ強化
  • 印刷用PDFダウンロード
